配置自定义域名

如果你没有域名，且是白嫖党 (而且是经费不足的那种) ，下面这一段你可以绕过了

前置准备

  • 一个域名(需要购买)

    这里推荐几个买域名的地方

    • 腾讯云DNSPod

      • 优点:价格第一年便宜，能低至7-8元的

      • 缺点:国内的要实名，而且实名审核要一定时间

        P.S. 像我们这种个人建议别买.cn域名，不是说.cn域名不好，而是因为有可能通过whois查出你的一些个人信息

        我相信看到这里一些人肯定想去查了，先说好了，要是出什么问题我可不负责XD~~

    • GoDaddy

      • 优点:国外的不需要实名

      • 缺点:价格相对贵一点,不支持微信支付

  • 一个在GitHub Pages部署好的网站

    Tip:要是没有的话可以按照我之前的文章来搭一个个人网站

快乐de配置过程

DNS解析方面

首先打开你域名DNS解析记录的控制台

没有链接提供! 各个平台控制台的位置不一样 ;<

然后新建一条CNAME记录

CNAME类似于你域名的一个别名,如果域名A建立了一条CNAME记录指向域名B,你请求A就会去请求B

现在我们看向上图中右边两个框框，又到了做填空的时候呐!

  • Name(名称)填你想要的子域名或者是@

    举个糖炒板栗(逃: 如果你的域名是nanosec.site,你这里输入@别人访问nanasec.site的时候是这个网站,如果你填了blog,别人访问blog.nanosec.site是你的网站

  • Target(目标)填你GitHub Pages哪个网站的域名，一般是你的用户名.github.io,如果你忘了你的用户名 (这什么有大病的如果) 你可以去下面的评论区发条评论，然后你的用户名就会显示出来啦 : 3

GitHub方面

首先登录到GitHub，然后打开你网站的那个仓库的设置的Pages栏

如果你找不到仓库了,你可以去下面的评论区发条评论，然后点击你的用户名，你的个人界面就打开了~~

然后在上图中的Custom domain下面的框中填写你之前配置好的域名

点击Save,等它检测完成….

看到上面有一行绿色的英文Your site is published at https://你的域名/就可以吃西瓜啦~(≧▽≦)/~

之后别人访问你的域名就可以打开你自己网站了

CloudFlare免费CDN配置

进入CloudFlare官网，选择免费的那个计划

然后就是日常的注册/登录操作

进入你个人用户的面板->Websites(网站)了之后,点击Add A Site(添加网站)

然后输入你的域名,点击Add Site(添加网站)

之后选择Free(免费)的那个计划，

接下来的操作有了CloudFlare官方详细的提示就是有手就行的事情啦
